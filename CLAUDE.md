This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What is Florita

Florita is a French-language web app for occupational therapists (ergothérapeutes). It converts raw field notes dictated during home visits into structured, funding-body-specific clinical reports (comptes rendus). Reports are generated by Claude AI via a Supabase Edge Function with streaming, then saved to Supabase and can be exported as Word (.docx) files.

## Running locally

Serve the static frontend:
```
node _serve.js
# Opens at http://localhost:8732
```

Deploy/test the Edge Function locally (requires Supabase CLI):
```
supabase start
supabase functions serve generate-cr
```

Deploy the Edge Function to production:
```
supabase functions deploy generate-cr
```

The Edge Function requires the `ANTHROPIC_API_KEY` environment variable set in the Supabase project secrets.

## Architecture

The app is a zero-build static frontend — no bundler, no framework. All pages are plain HTML files served directly. There is no `package.json`.

**Page flow:**
1. `login.html` — Supabase email/password auth (sign up + sign in)
2. `dashboard.html` — post-login landing page
3. `formulaire.html` + `formulaire.js` — form to collect patient data, notes, measures, trame selection; writes to `localStorage` under `florita-cr-data` and redirects to `index.html`
4. `index.html` + `script.js` — reads `localStorage`, calls the Edge Function with streaming, renders the CR in a `contenteditable` div, saves the result to Supabase `generated_crs` table, and exports to Word via the `docx` UMD bundle
5. `historique.html` — reads `generated_crs` from Supabase, renders CR cards with preview/download/delete actions
6. `trames.html` — UI for browsing/editing the report templates (trames)

**Auth:**
`auth.js` is a script injected into protected pages. It initialises the Supabase client as `window.floritaSb` and redirects unauthenticated users to `login.html`. The Supabase anon key is intentionally public (it is a `sb_publishable_*` key).

**Trames (report templates):**
`trames.js` exports three named template strings — `TRAME_VAD_CONTENU` (home visit / MAIF), `TRAME_TEL` (phone assessment), `TRAME_ACTION` (action plan). The form in `formulaire.html` lets users pick one; it is passed to the Edge Function as `trame`.

**Edge Function (`supabase/functions/generate-cr/index.ts`):**
- Receives `{ destination, notes, measures, trame, sexe }` via POST
- Builds a system prompt in `buildSystemPrompt()` with destination-specific argumentation rules (`DEST_RULES`)
- Calls `claude-sonnet-4-6` with `stream: true`, `max_tokens: 8000`, `temperature: 0.3`
- Streams raw text back to the browser (SSE → plain text passthrough)
- `verify_jwt = false` — no Supabase JWT check; the Anthropic key is the only secret

**Markdown subset used in generated text:**
- `## Section title` → `<h3>` in the viewer
- `**bold**` → `<strong>`
- `§§text§§` → `<mark class="hl">` (yellow highlight, used to flag sections needing completion)
- `---` → `<hr>`

**Word export:**
Client-side only, using `docx@8.5.0` loaded from unpkg. The `exportWord()` function in `script.js` (and a duplicate in `historique.html`) walks the rendered DOM nodes and maps them to `docx` `Paragraph`/`TextRun` objects.

## Key domain rules (from the AI system prompt)

- The generated CR must always start with a BROUILLON disclaimer mentioning ANFE April 2026 recommendations.
- Never use "chutogène" — use "risque de chute". Never use "glissance" — use "surface glissante".
- For insurance destinations ("Assurance dommages corporels", "Protection juridique"), never assert legal imputability — use conditional phrasing only.
- Sections with no data must use `§§À compléter par l'ergothérapeute§§`.
- Bold is reserved for section titles only in the body text.
